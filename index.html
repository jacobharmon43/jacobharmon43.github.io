Data Sorter

A useful tool for visualized the differences in sorting algorithms based on how they travel, how long they take, and how many instructions (Comparisons and Swaps) they call. Temporarily finished. Still To Do: Merge sort if I can get unity to run recursion

Results of project: In any time sensitive scenario, such as real-time performance, quick-sort takes the cake. It is so much faster than every other sort, even in small data set scenarios. For space sensitivity, insert sort reigns supreme of these, consistently taking a bit longer than quick sort, but having approximately 2/3rds as many instructions ran. It is also significantly easier to code and implement than quick sort. Aside from the data easily extrapolated from this small program, I also learned a lot about each individual sorting algorithm, from their code, to their naming history. One such fact that I learned is that gnone sort is named so, because the example scenario of the sorting algorithm is a gnome tending to his garden plants, grabbing the smaller plants one by one and moving them towards the front.
